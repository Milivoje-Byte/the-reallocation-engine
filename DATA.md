# Data

This repo ships **small samples** so the workflow runs the moment you clone it.
The **full datasets are large and gitignored** — they're public and regenerable,
so there's no reason to carry hundreds of MB in git history.

## How it works
- `*/sample/` folders hold trimmed data (a few hundred KB) — enough to run and
  verify the pipeline end to end.
- The full files are downloaded/regenerated by the scripts below.
- Files over **25 MB** are auto-ignored (see "Size policy"). The big files stay
  on your disk after you fetch them; they just never get committed.

## Get the full datasets

| Full file (gitignored) | Sample shipped | How to get the full version |
|---|---|---|
| `data/sec/form-d/processed/companies-sec-*-d.json` | `…/processed/sample/*.sample.json` | `python3 scripts/sec/download-form-d-quarters.py --quarters 2025Q2 2025Q3 2025Q4 2026Q1 --user "Your Name you@email.com"` then the processing step (see `scripts/sec/`). Source: SEC EDGAR Form D. |
| `data/bls/db-30-2-text/Work Context.txt` | `…/db-30-2-text/sample/Work Context.sample.txt` | Download the O*NET database (text) from onetcenter.org/database, or run `scripts/bls/`. |
| `data/bls/zip/db_30_2_excel.zip` | *(none — binary archive)* | Download the O*NET database (Excel) from onetcenter.org/database. |

## Size policy
- **Ignore-large sweep:** `scripts/gitignore-large.sh --mb 25 --apply --untrack`
  finds files over the limit, adds them to `.gitignore`, and stops tracking them
  (keeping them on disk). Run it any time the data grows.
- **Commit guard (the "by default"):** `.githooks/pre-commit` blocks any commit
  containing a file ≥ 50 MB. **Enable it once per clone:**
  `git config core.hooksPath .githooks`
- **To push a big file on purpose:** `git add -f path/to/file` and, if the guard
  trips, `git commit --no-verify`.

> Note: gitignoring affects the future only. Large files already in git history
> stay there; shrinking that requires a separate rewrite (git filter-repo / BFG).
